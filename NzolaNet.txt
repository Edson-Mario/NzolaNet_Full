### ARTEFATO 5: RULES.md
Regras de código e constraints:
```markdown
# RULES.md

## R001: Estrutura de Arquivos
- Todo arquivo deve ter propósito único (Single Responsibility)
- Nomear arquivos no padrão: [dominio].[camada].[ext] (ex: user.service.ts)
- Máximo 300 linhas por arquivo. Se exceder, dividir.

## R002: Nomenclatura
- Variáveis: camelCase, descritivas, sem abreviações (exceto id, db, api)
- Funções: verbo + substantivo (ex: getUserById, validateEmail)
- Classes: PascalCase, substantivo (ex: UserRepository)
- Constantes: SCREAMING_SNAKE_CASE

## R003: Clean Code
- Funções máximo 20 linhas
- Máximo 3 parâmetros por função (usar objeto se mais)
- Early returns obrigatórios
- Não usar var, apenas const/let
- Não usar any (TypeScript) ou tipagem implícita fraca

## R004: Testes
- Todo arquivo .ts/.js deve ter arquivo de teste correspondente
- Cobertura mínima: 80% das funções públicas
- Usar nome: [arquivo].test.[ext]

## R005: Documentação
- JSDoc para funções públicas complexas
- README.md em cada módulo explicando propósito
- CHANGELOG.md mantido a cada task concluída

## R006: Segurança
- Nunca commitar secrets (usar .env + .env.example)
- Validar todas as entradas de usuário
- Sanitizar dados antes de persistir
- Usar prepared statements para queries

## R007: Performance
- Lazy loading para rotas/componentes pesados
- Imagens otimizadas (WebP/AVIF)
- Bundle splitting quando aplicável
- Debounce/throttle em eventos frequentes
```

# Análise do Sistema NzolaNet

## 1. Visão Geral do Sistema

O sistema **NzolaNet** será uma aplicação web de rede social desenvolvida para permitir que utilizadores publiquem conteúdos, interajam através de bazes (reações), comentários e mantenham um perfil pessoal.

A aplicação será composta por:

* Gestão de Utilizadores
* Gestão de Publicações
* Gestão de Bazes
* Gestão de Comentários
* Feed de Notícias
* Sistema de Notificações
* Controlo de Privacidade

O sistema deverá funcionar em arquitetura em camadas:

* Frontend (Angular)
* Backend API (ASP.NET Web API)
* Base de Dados (SQL Server)

Também deverá utilizar:

* Controllers
* Services
* Repositories
* DTOs (Data Transfer Objects)

---

# 2. Funcionamento do Sistema

## 2.1 Fluxo Geral do Sistema

### 1. Registo e Autenticação

O utilizador cria uma conta fornecendo:

* Nome
* Email
* Palavra-passe
* Foto de perfil (opcional) 

Após o registo:

* O sistema valida os dados
* A palavra-passe é criptografada
* O utilizador é autenticado
* É criada uma sessão/token JWT

O utilizador também poderá:

* Recuperar senha
* Editar perfil
* Alterar foto
* Definir privacidade do perfil

---

### 2. Publicações

Depois de autenticado, o utilizador poderá:

* Criar publicações
* Inserir texto
* Adicionar imagem
* Adicionar vídeo
* Editar publicação própria
* Excluir publicação própria

Cada publicação terá:

* Nome do autor
* Foto do autor
* Data
* Texto
* Imagem (opcional)
* Vídeo (opcional)
* Quantidade de bazes
* Quantidade de comentários

As publicações serão exibidas:

* Em ordem cronológica
* No feed principal
* No feed de utilizadores seguidos

---

### 3. Sistema de Bazes

Os utilizadores poderão reagir às publicações.

Regras:

* Um utilizador só pode dar um baze por publicação
* Pode remover o baze
* O sistema atualiza automaticamente o contador de bazes
* O autor recebe notificação

---

### 4. Sistema de Comentários

Os utilizadores poderão:

* Adicionar comentários
* Editar comentários próprios
* Excluir comentários próprios
* Visualizar comentários da publicação

O sistema deverá:

* Associar comentário ao utilizador e publicação
* Permitir moderação por administrador
* Gerar notificação para o autor da publicação

---

### 5. Sistema de Seguidores

Os utilizadores poderão:

* Seguir outros utilizadores
* Deixar de seguir

Quando um novo seguidor for adicionado:

* O sistema cria uma notificação
* O feed poderá mostrar conteúdos dos utilizadores seguidos

---

### 6. Feed de Notícias

O feed será responsável por:

* Mostrar publicações recentes
* Mostrar publicações dos utilizadores seguidos
* Ordenar cronologicamente
* Atualizar dinamicamente

O sistema poderá utilizar:

* WebSockets
* SignalR

Para atualização em tempo real.

---

### 7. Notificações

O sistema deverá gerar notificações quando:

* Um utilizador recebe baze
* Um utilizador recebe comentário na sua publicação
* Um utilizador ganha seguidor ou perde seguidor

Cada notificação terá:

* Tipo
* Mensagem
* Data
* Estado (lida/não lida)

---
